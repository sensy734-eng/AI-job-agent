# PRD：AI 求职助手 Agent

## 1. 背景与目标

软件工程大三学生在投递 AI 相关实习时，常见问题是岗位 JD 读不透、简历无法针对岗位改写、面试准备缺少针对性。本产品通过“简历 + JD”输入，生成岗位匹配报告、简历优化建议和面试准备内容。

目标是交付一个可演示 MVP，同时能体现需求分析、系统设计、AI Agent、RAG 检索和产品指标意识。

## 2. 用户与场景

- 目标用户：准备投递 AI 应用开发、后端开发、算法/大模型应用实习的本科生。
- 场景一：岗位筛选。用户复制 JD，系统判断简历与岗位是否匹配，指出优势和短板。
- 场景二：简历改写。系统基于 JD 和简历证据生成 before/after 建议，不虚构经历。
- 场景三：面试准备。系统根据匹配短板生成问题，并对用户回答给出反馈。

## 3. 核心流程

1. 用户输入简历文本或上传文件。
2. 用户输入岗位 JD。
3. 系统解析简历和 JD，抽取技能、项目、职责、学历要求和关键词。
4. RAG 检索简历/JD/题库/反馈中的相关证据。
5. Agent 生成匹配评分、优势、短板、简历改写建议和面试题。
6. 用户对建议进行采纳、拒绝或修改。
7. 反馈进入后续检索上下文，形成迭代闭环。

## 4. 功能需求

- 简历解析：抽取姓名、求职方向、教育背景、技能、项目、经历、获奖。
- JD 解析：抽取岗位名称、职责、必备技能、加分技能、学历要求、关键词。
- 匹配评分：按技能匹配 35%、项目经验 30%、关键词 15%、学历要求 10%、风险控制 10% 计算。
- 证据引用：每条建议和问题需要尽量关联简历或 JD 片段。
- 简历优化：生成 section、before、after、rationale、evidence_ids。
- 面试准备：生成题目、考察点、分类，并支持回答评分。
- 反馈迭代：记录用户对建议的 accept/reject/revise。

## 5. 非功能需求

- 可演示：无模型 Key 时也能使用离线规则引擎跑通。
- 可扩展：保留 OpenAI-compatible Provider、LangGraph、pgvector 接入位置。
- 可追溯：生成建议不得虚构公司、项目、指标和经历。
- 响应速度：MVP 匹配报告 P95 小于 20 秒。
- 隐私：演示数据脱敏，不默认上传第三方服务。

## 6. MVP 边界

- 不做登录、岗位爬虫、自动投递、支付、企业后台。
- 不做复杂 ATS 排版导出，第一版只输出结构化改写建议。
- 不强依赖真实 LLM，默认离线规则，后续可切云端模型。
